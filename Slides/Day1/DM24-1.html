R Objects

  • R assignments and function outputs create objects
    • R takes some features from OOP languages, but is mostly functional in nature
    • Mostly useful for polymorphism
  • One of the following types:
    • Vectors
    • Matrices
    • Arrays
    • Data frames/Tibbles (and Lists)

Vectors

  • Vectors are the most basic object in R.

    • One dimensional
    • Can be accesed using the [ operator
    • They can be integer, numeric, logical, or character (no mixes)
    # Scalar
vec_one <- 4
vec_one[1] # 1-indexed!
    [1] 4
    # Vectors 
num_vec <- c(1, 9, 8)
int_vec <- c(1L, 9L, 8L)
chr_vec <- c("S","Oli","S")
bool_vec <- c(TRUE, FALSE)

Matrices

  • Generalization of vectors

    • Two dimensional
    • Can perform matrix operations, including multiplication (%*%), inversion (solve()), and transposition (t()).
    # Matrix (column major ordering!)
small_mat <- matrix(c(1:6), ncol = 2)
small_mat
         [,1] [,2]
[1,]    1    4
[2,]    2    5
[3,]    3    6
    num_vec %*% small_mat
         [,1] [,2]
[1,]   43   97

Arrays

  • Tensor storage for R

    • Three or higher dimensional
    • Slices can be implicitly transformed to vectors and matrices
    # Array
small_array <- array(c(1:12), dim = c(3, 2, 2))
small_array[1,2,1]
    [1] 4
    num_vec %*% small_array[,,1]
         [,1] [,2]
[1,]   43   97
    # num_vec %*% small_array[,,1, drop=FALSE] (Error!)

Lists

  • Most flexible storage type for R

    • Elements can be of different types and lengths
    • Can be named or un-named
    • Access elements by name (using $ or [[) or by location (using [[)
    # List
small_list <- list(first = 1:3, second = letters[1:4])
small_list[[1]] # small_list[["first"]] also works
    [1] 1 2 3
    small_list$second
    [1] "a" "b" "c" "d"

Data Frames

  • Lists with named elements of equal length

    • Can be thought of as a rectangular dataset
    • Typically uses factors to store character vectors
    • Exploit list nature to attach complex objects to observations

Data Wrangling II: Merging

Preliminaries

  • A Key is a variable (or a combination of variables) that uniquely identifies an observation in a table.
    • For example, year is a key in the turnout table, as it uniquely identifies each election in the dataset
  • You can define relations between the keys of two datasets (a.k.a. the source, or primary key, and the foreign key)
    • Can be used to merge (or join) datasets together.
      • Bring new variables into a dataset
      • Filter observations based on presence in another dataset; no new variables are appended to the returned tibble.

Types of mutating joins

 

Inner Join

  • With an inner_join(x, y), only observations that appear in both x and y datasets are returned
    • The primary key must appear in the y dataset, and the foreign key must appear in the x dataset for an observation with that key to be returned.
  • All variables in x and y are returned.
  • For non-key variables with the same names in x and y, suffixes are added to identify which dataset each variable came from.

Left & Right Join

  • A left_join(x, y) call returns all observations in x, and drops observations that only appear in y

  • Variables in y that do not exist in x are appended to x in the returned tibble.

    • For observation in y but not in x, a right_join(x, y) will use NA for values in variables in x that are not present in y.

Data Wrangling III: Tidying Data

Principles of Tidy Data

  • Each variable forms a column.
    • Columns do not contain multiple (types of) variables
  • Each observation forms a row.
    • Think of each row as your unit of analysis
  • Each type of observational unit forms a table.
    • All observations in a table (tibble or data frame) relate to the same class of units
  • (Learn more about these principles: https://bit.ly/TidyDataPaper)
  • Datasets in the wild are rarely tidy…

Data Wrangling IV: Piping (or putting it all together)

Piping multiple operations

  • One of the most powerful features of the tidyverse grammar is the ability to daisy-chain data management operations together
    • “Read data, then select these variables, then drop cases such that this is true, then merge to this other dataset…”
  • This can be easily accomplished using the the pipe operator, %>% (or |> since R 4.1).
    • In programming, piping consists of chaining operations from left to right, such that the output of an operation becomes the input of the subsequent operation
  • The vast majority of functions in the tidyverse (particularly those in dplyr) can be part of a pipe

Piping multiple operations

Exploring Data I: Numerical Summaries

Working with groups

  • Subgroup analysis can help us understand differences and similarities across classes of units
    • Exploratory steps can be taken using the group_by() and summarize() functions in dplyr (or the .by argument)
  • %>% group_by(x,y,z) takes a tibble, generates “subsets” based on unique combinations of values of (categorical) x, y, and z, and then performs subsequent operations on these subsets

Working with groups

Summarizing within groups

  • Operating on data subgroups is useful, but typically we are interested in summaries within groups
    • Summary statistics (mean(), median(), sd(), max(), etc.)
    • Counts (total n(), nr. of distinct elements n_distinct(), etc.)
    • …
  • summarize() reduces data to a single summary value
    • When used after a group_by() call, it does so within each derived subset

Summarizing within groups
